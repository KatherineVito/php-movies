# Reglas Git

- El nombre de la rama debe describir los cambios (ej: `add-movie-support`, `fix-category-validation`).
- Los mensajes de commit siguen las convenciones de `.claude/skills/commit-message/convenciones.md`.
- Los PRs se crean siempre en modo **draft**.
- Usar `.claude/skills/crear-pr/pr-template.md` como base para el body del PR.
- Si hay número de ticket, incluirlo en el mensaje de commit: `COS-XX: descripción`.
